Get Started → See Features
TypeScript import { Engram } from "@engram/sdk";

const engram = new Engram({ url: "http://localhost:4200" });

// Store what the agent learns
await engram.store("User prefers dark mode", {
  category: "decision",
  importance: 8,
});

// In a new session — recall relevant context
const ctx = await engram.recall("setting up editor");
// → Returns the dark mode preference automatically
Features

Everything agents need to remember

One server. One SQLite file. No external services required. Native TypeScript support via Node.js 22+.

🧠

Semantic Search

1024-dim embeddings via BGE-large-en-v1.5 with raw ONNX inference. Find memories by meaning, not keywords. No OpenAI key required.

🔗

Auto-Linking

New memories automatically connect to related ones above 0.75 similarity, forming a knowledge graph you can visualize and traverse.

🔄

Versioning

Update memories without losing history. Full version chain preserved. Only the latest version surfaces in search and recall.

🧹

Auto-Deduplication

Detects near-identical memories and merges them. Source count tracks how often something was mentioned — signal, not noise.

⏰

Auto-Forget

Set memories to expire. A background sweep runs every 5 minutes. Low-importance memories fade. Critical ones persist forever.

🔍

Fact Extraction

Optional LLM-powered extraction of static facts from conversations. Supports any OpenAI-compatible API. Facts become high-priority recall candidates.

👥

Multi-Tenant

Isolated memory per user with API keys, rate limiting, and scoped queries. Each user sees only their own memories.

📦

Spaces

Organize memories into named collections. Switch between projects, domains, or contexts with a single header.

📊

Graph Visualization

WebGL galaxy-style graph of your memory space. Explore connections, search, create, edit — all from the browser.

🕸️

Graph Intelligence

Memories form a knowledge graph with relationship inference. Auto-compute centrality, shortest paths, and community detection. Understand what matters.

📖

Episodic Memory

Store conversation episodes as narratives — what happened, what decisions were made, what was produced. Facts link back to their source episode for full context.

🔧

MCP Server

Native Model Context Protocol integration for Claude Desktop, Cursor, and Windsurf. Expose Engram as AI-native tools with type-safe interfaces.

How It Works

Five-layer contextual recall

Recall doesn't just search — it assembles the optimal context window from multiple strategies.

🏗️

Static Facts

Always included. Things that are persistently true about the user. Extracted automatically from conversations.

🎯

Semantic Match

1024-dim embedding similarity to the current query. The most relevant memories float to the top.

📖

Episodic Context

Conversation episodes provide narrative context — what happened, why, and what decisions were made. Facts link back to their source episode.

🔗

Linked Memories

Graph expansion pulls in related memories from the knowledge graph. Context grows outward from what was found.

🕐

Recent Activity

Latest memories provide temporal context. The agent knows what just happened.

Under the Hood

Built different

No wrappers. No cloud APIs. Raw inference on your hardware.

🧠

BGE-large-en-v1.5 (1024-dim)

Raw ONNX inference via onnxruntime-node with a hand-written BERT WordPiece tokenizer. No @huggingface/transformers, no Python, no wrappers. 512-token context, quantized INT8, sub-200ms per embedding.

📖

Episodic + Semantic Memory

Two memory systems working together. Atomic facts for precise lookup. Conversation episodes for narrative context — what happened, why decisions were made, what the AI produced. Facts link back to their source episode.

🧪

FSRS-6 Spaced Repetition

Power-law forgetting curve with 21 trained weights from millions of Anki reviews. Dual-strength memory model (storage + retrieval). Every memory access is an implicit review. Memories that matter persist; noise fades naturally.
