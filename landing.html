Give your agents persistent, searchable, self-organizing memory. Store what they learn. Recall it when it matters. No external APIs required.

Get Started → Try Live Demo
TypeScript import { Engram } from "@engram/sdk";

const engram = new Engram({ url: "http://localhost:4200" });

// Store what the agent learns
await engram.store("User prefers dark mode", {
  category: "decision",
  importance: 8,
});

// In a new session — recall relevant context
const ctx = await engram.recall("setting up editor");
// → Returns the dark mode preference automatically

Also in v5.0

Cognitive science meets vector search

Two major upgrades that change how Engram thinks about memory.

🧠

FSRS-6 Spaced Repetition

Replaces exponential decay with a 21-parameter power-law forgetting curve trained on millions of real reviews. Memory stability grows with each access — just like human memory.

R = (1 + t/S)^(-w₂₀) Power-law decay
⚡ Storage strength — never decays
📉 Retrieval strength — power-law decay
🔍

Native Vector Search

Built on libsql with FLOAT32(384) vector columns and ANN indexing. Search is O(log n) via vector_top_k() instead of brute-force loops.

vector_top_k(idx, vec, 25) ANN index
📊 Embeddings still generated locally
🚀 No external API needed
Features

35+ features. One libsql file.

Zero external dependencies. FSRS-6 spaced repetition, native vector search, dual-strength memory. Ships in a single TypeScript file.

Core Memory
🧠

Semantic + Full-Text Search

384-dim embeddings with native libsql vector indexing (FLOAT32 + ANN). O(log n) vector_top_k search combined with FTS5 keyword matching. No external API required.

🔗

Auto-Linking & Knowledge Graph

New memories automatically connect to related ones via embedding similarity. Typed relationships: updates, extends, contradicts, caused_by, prerequisite_for.

🔄

Versioning & Dedup

Update memories without losing history. Full version chain preserved. Near-duplicates detected and merged automatically. Source count tracks confirmation frequency.

📊

FSRS-6 Spaced Repetition

21-parameter power-law forgetting curve (ported from open-spaced-repetition). Every access is an implicit review building stability. Dual-strength model tracks storage strength (never decays) + retrieval strength (power-law decay).

🏷️

Tags, Episodes & Spaces

Organize memories with tags, group into episodic sessions, and isolate by named spaces. Filter and scope all queries.

⏰

Auto-Forget

Set memories to expire with TTL. Background sweep every 5 minutes. Low-importance memories fade. Critical ones persist forever.

Intelligence Layer
🔍

Fact Extraction & Auto-Tagging

LLM-powered extraction of atomic facts from conversations and text. Automatic category classification, importance rating, static/dynamic detection, and keyword tagging.

⚡

Contradiction Detection

Scans for conflicting memories. Optional LLM verification eliminates false positives. Resolve by keeping one, both, or LLM-powered merge into a single truth.

💭

Reflections

Periodic meta-analysis over day, week, or month. Identifies themes, progress, patterns, and unresolved items. Reflections become searchable memories themselves.

🧬

Derived Memories

Inference engine finds patterns across memories and generates new facts that aren't explicitly stated. Each inference cites its sources with confidence scores.

🗜️

Auto-Consolidation

Large clusters of related memories get summarized into a single dense memory. Originals archived, links preserved. Keeps the memory space clean.

🏆

LLM Reranker

Search results reranked by an LLM for semantic precision. Candidates retrieved by embedding similarity, then filtered by a language model that understands nuance.

Retrieval
🎯

Smart Context Builder

Token-budget-aware RAG assembly. Four-phase pipeline: static facts → semantic search → linked expansion → recent. Three strategies: balanced, precision, breadth.

⏪

Time-Travel Queries

Query what the agent knew at any point in time. Version-chain-aware — finds the correct memory state as of any timestamp, with optional semantic search.

📋

Prompt Templates & Packing

Generate ready-to-use prompt context blocks. Pack memories into token-budgeted strings optimized for any LLM context window.

Platform
👥

Multi-Tenant

Isolated memory per user with scoped API keys, rate limiting, and per-key permissions. Each user sees only their own memories.

📬

Webhooks & Digests

Event webhooks on memory changes. Scheduled digest delivery (hourly/daily/weekly) via HMAC-signed webhooks with stats, new memories, and reflections.

🔄

Sync, Import & Ingest

Cross-instance sync via change feeds. Import from Mem0 and Supermemory. Ingest URLs or text blobs — Engram extracts facts and stores them as linked memories.

📊

Graph Visualization

Built-in WebGL galaxy graph. Explore connections, search, create, edit, archive — all from the browser. Category filters, keyboard shortcuts, auto-refresh.

🧩

Entities & Projects

Track named entities (people, servers, tools) and projects. Link memories to entities and projects. Scoped search within any entity or project.

🛠️

MCP Server & CLI

Model Context Protocol server for IDE integrations. CLI for terminal workflows. Both ship with the same binary — no extra install.

How It Works

Four-layer contextual recall

Recall doesn't just search — it assembles the optimal context window from multiple strategies.

🏗️

Static Facts

Always included. Things that are persistently true about the user. Extracted automatically from conversations.

🎯

Semantic Match

Embedding similarity to the current query. The most relevant memories float to the top.

⭐

High Importance

Critical memories (importance ≥ 7) surface regardless of query similarity. The agent never forgets what matters.

🕐

Recent Activity

Latest memories provide temporal context. The agent knows what just happened.
