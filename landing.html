const mem = new Engram({ url: "http://localhost:4200" });

Features

Everything agents need to remember

One server. One SQLite file. Runs on Node.js 22+ or Bun. No external services required.

Smart Memory
⏱️

FSRS-6 Spaced Repetition

21-parameter algorithm trained on millions of Anki reviews. Power-law forgetting curves. Every access is an implicit review — stability grows over time.

💪

Dual-Strength Model

Bjork & Bjork (1992): storage strength never decays, retrieval strength resets on access. Combined retention score: 0.7 × retrieval + 0.3 × storage.

🔄

Versioning

Update memories without losing history. Full version chain preserved. Only the latest version surfaces in search and recall.

🧹

Auto-Deduplication

Detects near-identical memories and merges them. Source count tracks how often something was mentioned — signal, not noise.

⏰

Auto-Forget / TTL

Set memories to expire with forget_after. Background sweep every 5 minutes. Critical memories persist forever.

⚡

Contradiction Detection

LLM-verified conflict scanning eliminates false positives. Resolve by keeping one side, both, or merging into a single memory.

Intelligence Layer
🧠

Hybrid Semantic + FTS5

MiniLM 384-dim embeddings (runs locally via ONNX) combined with SQLite FTS5 full-text search. Meaning and keywords — no OpenAI key required.

🔍

Fact Extraction & Auto-Tagging

LLM extracts static facts from conversations. Auto-tags with keywords, classifies importance, detects relationship types (updates, extends, contradicts, caused_by).

💬

Conversation Extraction

Feed raw chat logs to /add. Get structured memories out. Works with any agent framework or raw message arrays.

🌐

URL Ingest

Point /ingest at a URL or paste raw text. Facts extracted and stored automatically with source tracking.

💭

Reflections

On-demand meta-analysis detects themes, patterns, and progress across your memory space. Reflections become searchable memories.

🧬

Derived Memories & Consolidation

LLM inference finds patterns across clusters and generates new knowledge. Auto-consolidation compresses large clusters into dense memories.

Developer Platform
🔧

MCP Server

Native Model Context Protocol — JSON-RPC 2.0 stdio. Works with Claude Desktop, Cursor, and Windsurf out of the box. 8 tools, 3 resources.

⌨️

CLI

engram store, engram recall, engram search and more. Full access to every feature from your terminal. Stdin piping supported.

👥

Multi-Tenant + RBAC

Isolated memory per user. Role-based access: admin, writer, reader. Per-key rate limiting, scoped permissions, API key management.

📬

Webhooks & Digests

Event hooks on every mutation with HMAC signing. Scheduled memory digest delivery. Built-in SSRF protection on webhook URLs.

🔄

Cross-Instance Sync

Pull changes since a timestamp, push to another instance. Keep multiple Engram deployments in sync across environments.

📋

Audit Trail & Structured Logging

Every mutation logged — who, what, when, from where, with request IDs. Queryable via /audit. JSON logs throughout.

Visualization & Organization
🌌

WebGL Galaxy Graph

Explore your entire memory space as an interactive galaxy at /gui. Click memories, trace connections, search, create, and edit in the browser.

🕸️

Knowledge Graph

Auto-linking via cosine similarity with typed edges. Graphology-powered centrality, community detection, and shortest path traversal.

📦

Spaces, Tags & Episodes

Named memory spaces scoped via header. Tag anything. Group interactions into episode sessions for temporal context.

🧩

Entities & Projects

Track people, servers, tools, and projects as first-class objects with scoped search and relationship tracking.

📥

Review Queue / Inbox

Auto-detected memories land in inbox for review. Approve, reject, or edit before they enter recall. Bulk operations supported.

⏪

Time-Travel Queries

Query what you knew at any past moment. Debug decisions, audit context drift, understand how your agent's worldview evolved.

How It Works

The memory lifecycle

From store to recall — what happens to every memory.

01
💾

Store

Content embedded via local MiniLM ONNX (384-dim), stored in libsql with FTS5 full-text indexing.

02
🔗

Auto-Link

Compared to existing memories. Above 0.7 cosine similarity, typed relationship edges are created automatically.

03
⏱️

FSRS-6 Init

Stability, difficulty, storage + retrieval strength initialized. Power-law forgetting curve begins tracking retrievability.

04
🔍

Fact Extract

If LLM configured: static facts extracted, keywords tagged, importance scored, relationships to existing memories detected.

05
🎯

Recall

Four strategies combined: static facts (always), semantic matches, high-importance (FSRS-weighted), recent activity.

06
📈

Review

Every recall is an implicit FSRS-6 "Good" review. Stability builds. Frequently recalled memories can reach months of retention.

07
💪

Dual Decay

Storage strength accumulates. Retrieval strength decays via power law. Combined: 0.7 × retrieval + 0.3 × (storage/10).

08
⚡

Contradiction

Conflicts detected and LLM-verified. Resolved by keeping one, both, or merging. False positives eliminated automatically.

09
🗜️

Consolidate

Large related clusters summarized into a single dense memory. Originals archived, graph links preserved.

10
💭

Reflect

Periodic meta-analysis generates insights about themes, progress, and patterns. Reflections become searchable memories.

MCP Server

Works with your AI tools

Native Model Context Protocol — drop Engram into Claude Desktop, Cursor, or Windsurf in under a minute.

JSON claude_desktop_config.json
{
  "mcpServers": {
    "engram": {
      "command": "node",
      "args": ["path/to/engram/mcp-server.mjs"],
      "env": {
        "ENGRAM_URL": "http://localhost:4200",
        "ENGRAM_API_KEY": "eg_your_key"
      }
    }
  }
}
